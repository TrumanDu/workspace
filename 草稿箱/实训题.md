# 实训考试题（三选一）

## 考试说明

-   考生需从以下 **三个题目中任选一个完成**
-   实训时间：建议 4–6 小时
-   允许查阅官方文档，但禁止直接复制现成完整项目
-   重点考察：工程能力、系统理解能力与实际落地能力

---

## 题目一：实现一个 MCP Server（天气查询）

### 一、背景说明

MCP（Model Context Protocol）用于让大模型以标准化方式调用外部工具或服务。本题要求考生自行实现一个 MCP Server，并在 Cherry Studio 中成功调用，实现实时天气查询功能。

---

### 二、任务要求

1. **自行实现一个 MCP Server**

    - 可使用任意语言（推荐：Node.js / Python）
    - 符合 MCP Server 基本规范（tools / schema / invocation）

2. **实现天气查询工具**

    - 调用以下公开天气 API：

        ```
        https://api.open-meteo.com/v1/forecast?latitude=31.467&longitude=104.678&current_weather=true
        ```

    - 解析返回结果，提取关键天气信息

3. **在 Cherry Studio 中完成调用**

    - 成功注册 MCP Server
    - 能通过自然语言触发天气查询
    - 模型返回的结果应为清晰、可读的自然语言描述

---

### 三、交付物

-   MCP Server 源代码
-   MCP 工具定义（schema）
-   Cherry Studio 调用截图或录屏
-   简要说明文档（实现思路 + 启动方式）

---

### 四、评分要点（参考）

-   MCP 协议理解与实现正确性（30%）
-   天气 API 调用与数据解析（25%）
-   Cherry Studio 集成效果（25%）
-   返回结果的可读性与稳定性（20%）

## 题目二：搭建 Dify 并创建私有知识库工作流

### 一、背景说明

Dify 是一款用于构建 LLM 应用的开源平台。本题要求考生完成 Dify 的部署，并基于私有知识库构建一个可正常检索和回答问题的工作流。

---

### 二、任务要求

1. **成功搭建 Dify**

    - 支持本地部署或服务器部署
    - 可正常访问 Web 管理界面

2. **创建私有知识库**

    - 导入至少 3 条以上结构化或非结构化内容
      （如：文档、Markdown、文本说明等）
    - 确保知识可被成功索引与检索

3. **创建基于私有知识库的工作流**

    - 工作流需使用知识库作为数据来源
    - 能正确获取检索结果
    - 对检索结果进行加工，输出自然语言回答
      （禁止直接原样返回片段）

4. **验证效果**

    - 提供示例问题
    - 展示模型基于知识库给出的回答

---

### 三、交付物

-   Dify 部署说明或环境截图
-   知识库内容示例
-   工作流配置截图
-   问答效果示例
-   简要设计说明

---

### 四、评分要点（参考）

-   Dify 部署与配置完整性（25%）
-   知识库构建与检索效果（30%）
-   工作流设计合理性（25%）
-   回答的自然语言质量（20%）

## 题目三：使用 LangGraph 构建最小化智能体工作流

### 一、背景说明

LangGraph 是用于构建多步骤、可控智能体流程的框架。本题要求考生使用 LangGraph 构建一个**最小可运行的智能体工作流**，体现状态流转与节点协作。

---

### 二、任务要求

1. **使用 LangGraph 框架**

    - 基于官方推荐方式构建 StateGraph
    - 至少包含：

        - 状态定义（State）
        - 一个或多个节点（Node）
        - 明确的状态流转逻辑

2. **实现最小可运行智能体**

    - 能接收用户输入
    - 经至少一个中间节点处理
    - 输出最终结果

3. **体现智能体流程思想**

    - 节点职责清晰
    - 状态更新逻辑明确
    - 非简单“一次调用即返回”

---

### 三、交付物

-   LangGraph 示例代码
-   状态与节点说明
-   运行示例（输入与输出）
-   简要设计说明

---

### 四、评分要点（参考）

-   LangGraph 基本使用正确性（30%）
-   状态设计与流转清晰度（30%）
-   智能体流程合理性（20%）
-   代码可读性与稳定性（20%）
